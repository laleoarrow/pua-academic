# Grant Revision 味道方法论 — Agent 行为约束

> 切换到 Grant Revision 味道时自动加载。这些不是口号，是可执行的行为指令。

## 核心行为约束

### 1. 逐条回应原则
Grant Revision 的核心规则是 point-by-point response。Reviewer 的每一条意见都必须单独回应，编号对应，不可遗漏。回应格式：引用 Reviewer 原话 → 说明修改内容 → 指出修改位置（页码、行号）→ 如有新增实验，说明实验结果。漏掉一条意见 = Revision 失败。

### 2. 逻辑一致性检查
Significance、Innovation、Approach 三者必须逻辑一致。Significance 提出的问题，Innovation 提供的独特价值，Approach 提供可行的路径。三者脱节是 Grant 被拒的常见原因。每次修改都要检查整份 Grant 的逻辑链条。

### 3. Preliminary Data 充分性原则
Preliminary Data 是 Feasibility 的核心证据。Reviewers 常见的质疑是"preliminary data 不足以支持 proposed experiments"。每项 Specific Aim 都需要有对应的 preliminary data 支撑。Preliminary data 不是结果展示，是可行性证明。

### 4. Alternative Approaches 必备原则
每个 Specific Aim 必须有 Alternative Approaches。如果主要方法失败，Plan B 是什么？Reviewers 会问这个问题，你不提前准备就是送分题变送命题。Alternative 不是"换一个方法"，是"换一个思路解决问题"。

### 5. Timeline 和 Milestone 现实性原则
Grant Timeline 需要现实。如果 Reviewers 认为 timeline 不现实，会影响 Feasibility 评分。每个 milestone 需要具体、可量化、有时间节点。"第一年完成 Aim 1" 太模糊，"Year 1 Q1-Q2 完成 X，Q3-Q4 完成 Y，产出 Z" 才是合格的 milestone。

## 反面行为（碰了就触发压力升级）
- 漏掉 Reviewer 的某条意见不回应 → P7 严厉警告
- Significance/Innovation/Approach 逻辑脱节 → P7 要求重写
- Preliminary Data 不足但强行推进 → P7 施压
- 没有 Alternative Approaches 或敷衍了事 → P7 追问
- Timeline 明显不现实不调整 → P7 警告

## 自检清单（Grant Revision 味道特有）
- [ ] Reviewer 的每一条意见都有对应回应吗？
- [ ] Significance → Innovation → Approach 逻辑链条完整吗？
- [ ] 每个 Specific Aim 都有 Preliminary Data 支撑吗？
- [ ] 每个 Aim 都有 Alternative Approaches 吗？
- [ ] Timeline 和 Milestone 现实可执行吗？
- [ ] 字数限制是否满足？
- [ ] References 格式是否正确？

## Grant 结构模板

### Specific Aims Page 模板
```
TITLE: [Clear, Descriptive Title]

ABSTRACT (限制字数内):
[Background] → [Gap] → [Hypothesis] → [Aims] → [Impact]

SPECIFIC AIMS:
Overall Hypothesis: [One sentence]

Aim 1: [Title]
- Hypothesis
- Approach (2-3 句)
- Expected Outcome
- Potential Pitfalls & Alternatives

Aim 2: [Title]
- Hypothesis
- Approach
- Expected Outcome
- Potential Pitfalls & Alternatives

Aim 3: [Title] (如有)

IMPACT:
[Scientific Impact] → [Broader Impact]
```

### Research Strategy 模板
```
A. SIGNIFICANCE
1. Background and Gap
2. Our Preliminary Data
3. Impact

B. INNOVATION
1. What's New
2. Why We Are Uniquely Positioned
3. How This Advances the Field

C. APPROACH
For each Aim:
1. Rationale
2. Experimental Design
3. Methods
4. Expected Results
5. Potential Problems and Alternative Approaches
6. Timeline and Milestones
```

## Reviewer 意见应对策略

### Significance 质疑
```
问题: "Significance 不清楚"
应对:
1. 明确指出研究的 Gap
2. 说明解决这个 Gap 的重要性
3. 具体描述对领域的影响
4. 引用支持 Significance 的文献
```

### Innovation 质疑
```
问题: "Innovation 不足"
应对:
1. 与现有方法逐项对比
2. 强调 unique aspects
3. 说明为什么现有方法不够
4. 展示 preliminary data 证明 new approach 可行
```

### Feasibility 质疑
```
问题: "Feasibility 存疑"
应对:
1. 补充 Preliminary Data
2. 强调团队 Expertise
3. 展示成功的 pilot experiments
4. 提供 Alternative Approaches
```

### Investigator 质疑
```
问题: "Team expertise 不够"
应对:
1. 添加 Co-I 补充 expertise
2. 展示相关 publication record
3. 强调 collaborative track record
4. 提供 training plan（如需要）
```

## Agent 团队协作（PI/Postdoc 适用）

### Grant 写作分工
PI 负责 Significance、Innovation 和 Overall Strategy。Postdoc/Grad Student 负责 Approach 细节和 Preliminary Data 整理。Budget 和 Administrative 部分由 Research Administrator 处理。

### 内部评审流程
初稿完成 → Postdoc 自查 → PI 审阅 → Lab 内部 Mock Review → 修改 → 提交。每个环节都要预留时间。
