# Journal Submission 味道方法论 — Agent 行为约束

> 切换到 Journal Submission 味道时自动加载。这些不是口号，是可执行的行为指令。

## 核心行为约束

### 1. Cover Letter 战略原则
Cover Letter 是 Editor 的第一印象，决定是否送审。必须在第一段说清楚：研究解决了什么重要问题？为什么适合这个期刊？创新点是什么？不要重复 Abstract，要说 Abstract 说不了的"为什么"。Cover Letter 过关，才有后续。

### 2. Methods 可复现原则
Methods 是论文被质疑时最后的防线。必须包含：详细 protocol、试剂 catalog number、设备型号和参数、统计软件版本、数据处理步骤。Reviewer 应该能够根据 Methods 复现你的实验。Methods 写不清楚 = 可信度打折。

### 3. Figure 自解释原则
每个 Figure 应该能够独立讲述一个故事。Figure Legend 要完整：样本信息、实验条件、统计方法、样本量。读者只看 Figure 不看正文也能理解结果。Figure 是论文的"广告牌"。

### 4. Discussion 平衡原则
Discussion 不是 Results 的重复。需要：总结主要发现 → 与文献对比 → 解释差异 → 承认局限 → 展望未来。过度解读和自我贬低都是问题。平衡 = 在自信和谦逊之间找到科学家的位置。

### 5. Response to Reviewers 辩护原则
Response to Reviewers 是学术辩护，不是道歉信。同意的部分说明如何修改，不同意的地方用数据和逻辑反驳。"Thank you for the comment" 之后必须跟着实质性回应。Point-by-point，编号对应，页码行号标注。

## 反面行为（碰了就触发压力升级）
- Cover Letter 像 Abstract 重复 → P7 要求重写
- Methods 遗漏关键信息 → P7 追问
- Figure Legend 不完整 → P7 要求补充
- Discussion 过度解读或过于保守 → P7 施压
- Response to Reviewers 只道歉不辩护 → P7 警告

## 自检清单（Journal Submission 味道特有）
- [ ] Cover Letter 第一段说明为什么适合这个期刊？
- [ ] Methods 包含所有复现所需信息吗？
- [ ] 每个 Figure 都能独立理解吗？
- [ ] Discussion 平衡了发现、对比、局限、展望吗？
- [ ] References 格式正确吗？
- [ ] Supplementary Materials 完整吗？
- [ ] Author Contributions 和 Conflicts 声明准备好了吗？

## 论文结构模板

### Abstract 模板
```
Background (1-2 句): 为什么这个问题重要
Gap (1 句): 现有研究缺少什么
Methods (1-2 句): 我们做了什么
Results (2-3 句): 主要发现
Conclusion (1 句): 意义和影响

字数控制：通常 150-250 词
```

### Introduction 结构
```
第1段：Broad context - 为什么这个领域重要
第2段：Specific background - 已知什么
第3段：Gap - 不知道什么，为什么重要
第4段：Our approach - 我们怎么解决这个问题
第5段：Summary of findings - 主要发现（简短）
```

### Results 结构
```
每个 Figure/Experiment:
1. 开头句：这个实验的 purpose
2. 方法简介：怎么做的
3. 结果描述：发现了什么
4. 小结：说明什么

过渡句连接各部分
```

### Discussion 结构
```
第1段：Summary of main findings
第2-3段：与文献对比，解释差异
第4段：Mechanistic insights（如有）
第5段：Limitations
第6段：Future directions and conclusions
```

## Response to Reviewers 模板

### 标准回应格式
```
We thank the reviewer for this insightful comment.

Reviewer Comment: "[引用 Reviewer 原话]"

Response:
[我们的回应]
- 如果同意：We have revised the manuscript to include... (Page X, Line Y)
- 如果不同意：We appreciate the reviewer's perspective, but we respectfully note that... [数据和逻辑支持]
- 如果需要补充实验：In response to this comment, we have performed additional experiments... (Figure X)

Changes in manuscript:
Page X, Line Y: [原文] → [修改后]
```

### 常见 Reviewer 意见应对

#### "Sample size 不足"
```
Response:
We thank the reviewer for raising this important point. We have performed a power analysis which indicated that our sample size of N=XX provides XX% power to detect an effect size of XX at α=0.05. This power analysis has been added to the Methods section (Page X, Line Y).

Additionally, we note that our sample size is comparable to similar studies in the field [References].
```

#### "Missing control"
```
Response:
We appreciate the reviewer's suggestion. We have now included [additional control] in our experiments. The results show that... [描述结果]. This additional control strengthens our conclusion by... [解释为什么重要].

New data are presented in Figure X and described in the Results section (Page X, Line Y).
```

#### "Mechanism 不清楚"
```
Response:
We agree that understanding the mechanism is important. In this study, we focused on establishing the phenomenon and its functional significance. Preliminary mechanistic experiments suggest [简要描述], but a thorough mechanistic investigation is beyond the scope of this work. We have discussed this limitation and proposed future directions in the Discussion (Page X, Line Y).
```

## Agent 团队协作（PI/Postdoc 适用）

### 论文写作分工
PI 负责 Overall Story 和 Discussion 的科学判断。Postdoc 负责 Methods、Results 细节。图表制作由 First Author 负责，PI 审阅。

### 内部审核流程
初稿 → Co-authors 审阅 → PI 审阅 → Lab 内部预审 → 修改 → 投稿。确保投稿前所有 Co-authors 同意。
